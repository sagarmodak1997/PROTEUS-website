Our Mission

One framework, three engines, every integration.

Monte Carlo integration is the workhorse of collider phenomenology — but the best algorithm depends on the problem. PROTEUS (Phase-space and Resonance-aware Orchestrator for Tensor-accelerated Estimation via Unified Samplers) is a smart wrapper that examines your integrand, detects the available hardware, picks the right backend, and runs it in an isolated subprocess so TensorFlow and PyTorch never collide.

You write a function. PROTEUS handles backend selection, hardware detection, GPU memory management, multi-GPU assignment, CPU fallback, result aggregation, and uncertainty reporting. No reinvention — VegasFlow, i-flow, and MadNIS are called through their official APIs, exactly as their authors intended.

Architecture

Layered, modular, isolated.

A request flows through five layers — from your Python call to an isolated backend worker. Each layer is replaceable; each backend lives in its own subprocess. Supported native paths execute in GPU-enabled backends, while compatibility callback paths preserve Python/NumPy behavior.

Frontend
General · Physics · EFT Python APIs and a 7-tab Gradio web interface for general integration, HEP cross sections, hadronic processes, phase-space volumes, SMEFT operators, parameter gradients, and ONNX export.
Dispatcher
Smart backend selector Examines dimension, peakedness, and resonance structure. Routes low-dimensional jobs to VegasFlow, smooth high-dimensional jobs to i-flow, and mid-dimensional peaked or resonant jobs to MadNIS unless a backend is forced.
Worker
Subprocess isolation Each backend runs in a dedicated subprocess with its own framework — TensorFlow and PyTorch never share a process. Health-check pings, per-job timeouts, CUDA-wedge detection.
Backends
VegasFlow · i-flow · MadNIS · PDFFlow Three integration engines plus GPU-accelerated PDFs. Called through their official APIs — PROTEUS reimplements none of them.
Hardware
Multi-GPU orchestration Auto-detects NVIDIA CUDA, AMD ROCm/HIP, Apple Silicon Metal/MPS, or CPU-only systems. On multi-GPU hosts it offers per-backend assignment with memory fractions; without a supported GPU it falls back to CPU where backend dependencies allow.
Features

Built for production-grade phenomenology.

Every feature was driven by a real LHC-era use case: differentiable cross sections for likelihood fits, ONNX models for experiment software, phase-space generators with analytic validation.

⚡

Automatic backend selection

Diagnostic samples decide between Vegas, normalising flows, and neural multi-channel based on dimension and peakedness.

∂

Differentiable cross sections

Compute dσ/dθ via the Leibniz rule with central finite differences or tf.GradientTape — for any parameter, any process.

⚛

SMEFT operators

Warsaw-basis dimension-6 Wilson coefficient scans. Recovers SM exactly at Ci = 0; full interference + quadratic terms.

🔬

Hadronic processes

GPU-accelerated PDFs via PDFFlow. Drell-Yan and BSM resonance cross sections for LHC, Tevatron, and lepton colliders.

📐

RAMBO phase space

n-body Lorentz-invariant phase-space volumes Φₙ(s) for any number of final-state particles, with an analytic massless reference and pull validation.

📦

ONNX export

Export trained i-flow / MadNIS models to ONNX for C++ deployment in Delphes, full-sim, or experiment frameworks.

🎲

Automatic unweighting

Rejection unweighting and PyTorch-based neural unweighting with reported efficiency and per-event weights.

🔁

UQ ensembles

i-flow ensemble mean and variance for combined Monte-Carlo + machine-learning uncertainty quantification.

Demo

Five lines to the Z-pole cross section.

PROTEUS picks the backend, runs it in an isolated worker, and returns a typed result object with value, error, backend name, wall-clock time, and rich metadata. 

import proteus

r = proteus.compute_cross_section(
        process='ee_to_mumu',
        ecm=91.2,
        n_calls=500_000,
     )

print(f"σ = {r.value:.1f} ± {r.error:.1f} pb")
# σ ≈ 2010 ± 1 pb   (Z-pole, validated vs MadGraph)

import proteus, numpy as np

r = proteus.integrate(
        func    = lambda x: np.exp(-0.5*np.sum(x**2,axis=-1)),
        n_dim   = 5,
        bounds  = (-4, 4),
        n_calls = 200_000,
     )

print(r.value, r.error, r.backend)
# ≈ 98.9  ...  vegasflow
